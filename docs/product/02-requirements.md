# 02. 요구사항 정의 (Requirements)

## 1) 기능 요구사항
| ID | 기능 | 설명 | 우선순위 |
|---|---|---|---|
| FR-001 | 과제·시험 일정 등록/조회/수정/삭제 | 사용자가 과제와 시험 일정을 생성/조회/수정/삭제(CRUD)할 수 있어야 한다 | 높음 |

## 2) 비기능 요구사항
- 성능: 주요 화면 로딩 3초 이내
- 보안: JWT(Access 15분, Refresh 14일) 기반 인증, 비밀번호 bcrypt 해시 저장, 모든 인증 API 접근 제어 적용
- 안정성: 치명적 오류 발생 시 사용자에게 안내 메시지를 제공한다
- 접근성: 키보드만으로 주요 기능 접근 가능

## 3) 범위 제외 (이번 버전에서 하지 않는 것)
- 성적 분석/통계 기능은 제외

## 4) 완료 기준 (Definition of Done)
- [ ] 핵심 기능 동작 확인
- [ ] 에러 시 사용자 안내 문구 제공
- [ ] 테스트 시나리오 통과
