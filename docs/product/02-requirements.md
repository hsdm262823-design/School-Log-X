# 02. 요구사항 정의 (Requirements)

## 1) 기능 요구사항
| ID | 기능 | 설명 | 우선순위 |
|---|---|---|---|
| FR-001 | 과제·시험 일정 등록/조회/수정/삭제 | 사용자가 과제와 시험 일정을 생성/조회/수정/삭제(CRUD)할 수 있어야 한다 | 높음 |
| FR-002 | 통합 캘린더 보기 | 월/주/일 캘린더에서 과제·시험 일정을 한눈에 확인할 수 있어야 한다 | 높음 |
| FR-003 | D-day 알림 | 마감/시험일 기준으로 다가오는 일정을 D-day 형태로 보여줘야 한다 | 높음 |
| FR-004 | 과목별 진행률 체크 | 과목별로 과제 진행률(0~100%)을 확인하고 수정할 수 있어야 한다 | 높음 |
| FR-005 | 날짜 입력 분리 UI | 날짜 입력은 연/월/일 분리 입력(4자리/2자리/2자리)으로 제공하고, 월/일 한 자리 입력도 허용해야 한다 | 높음 |
| FR-006 | 날짜 기본값 자동 채움 | 일정 등록 폼의 날짜 기본값은 앱 실행일(당일)로 자동 채워져야 한다 | 중간 |

## 2) 비기능 요구사항
- 성능: 주요 화면 로딩 3초 이내
- 보안: 서버 입력값 검증, SQL Injection/XSS 방지, 비밀값 환경변수 관리 적용
- 안정성: 치명적 오류 발생 시 사용자에게 안내 메시지를 제공한다
- 접근성: 키보드만으로 주요 기능 접근 가능

## 3) 범위 제외 (이번 버전에서 하지 않는 것)
- 성적 분석/통계 기능은 제외
- 회원가입/로그인/JWT 인증 기능은 제외 (비로그인 단일 사용자 MVP)

## 4) 완료 기준 (Definition of Done)
- [ ] 핵심 기능 3개(통합 캘린더, D-day 알림, 과목별 진행률 체크) 동작 확인
- [ ] 에러 시 사용자 안내 문구 제공
- [ ] 핵심 시나리오 20건 수동 테스트 중 19건 이상 성공(95%)
