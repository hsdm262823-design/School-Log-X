# 03. 사용자 흐름 (User Flow)

## 1) 메인 흐름
앱 실행 → 홈 → 일정 확인/등록/수정 → 완료

## 2) 상세 단계
1. 시작 화면
2. 홈 화면
3. 기능 실행(과제/시험 일정 등록, D-day 확인, 진행률 수정)
4. 완료/저장

## 3) 예외 흐름
- 네트워크 오류 시: 오프라인/오류 안내 후 재시도 버튼 제공
- 입력값 오류 시: 필드별 오류 메시지 표시 + 저장 차단

## 4) 화면별 필요 정보
| 화면 | 사용자 행동 | 시스템 반응 |
|---|---|---|
| 시작 화면 | 앱 시작 후 홈 화면으로 이동 | 초기 데이터 로드 후 홈 화면 렌더링 |
| 홈 화면 | 과제/시험 일정 확인 및 새 일정 등록 | 저장 성공 시 목록 갱신 및 완료 안내 표시 |
| 홈 화면 | 다가오는 일정 D-day 확인 | 현재일 기준 D-day 계산 결과 표시 |
| 홈 화면 | 과목별 진행률 수정 | 수정 성공 시 과목 진행률 카드 즉시 갱신 |
