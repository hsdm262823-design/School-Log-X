# 03. 사용자 흐름 (User Flow)

## 1) 메인 흐름
앱 실행 → 로그인/회원가입 → 홈 → 일정 확인/등록 → 완료

## 2) 상세 단계
1. 시작 화면
2. 로그인/회원가입
3. 홈 화면
4. 기능 실행(과제/시험 일정 등록)
5. 완료/저장

## 3) 예외 흐름
- 로그인 실패 시: 오류 메시지 표시 후 재시도 유도
- 네트워크 오류 시: 오프라인/오류 안내 후 재시도 버튼 제공
- 입력값 오류 시: 필드별 오류 메시지 표시 + 저장 차단

## 4) 화면별 필요 정보
| 화면 | 사용자 행동 | 시스템 반응 |
|---|---|---|
| 시작 화면 | 앱 시작 후 로그인 화면으로 이동 | 인증 상태 확인 후 로그인 화면 렌더링 |
| 로그인 화면 | 이메일/비밀번호 입력 후 로그인 요청 | 자격 증명 검증 후 성공 시 홈 이동, 실패 시 오류 표시 |
| 홈 화면 | 과제/시험 일정 확인 및 새 일정 등록 | 저장 성공 시 목록 갱신 및 완료 안내 표시 |
