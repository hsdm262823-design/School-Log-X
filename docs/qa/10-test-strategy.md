# 10. 테스트 전략 (Test Strategy)

## 1) 테스트 목표
- 핵심 기능이 안정적으로 동작하는지 확인
- 수정 후 기존 기능이 깨지지 않는지 확인

## 2) 테스트 범위
- 단위 테스트: 함수 단위
- 통합 테스트: API + DB
- E2E 테스트: 사용자 시나리오 전체

## 3) 기본 시나리오
1. 일정 등록 시 목록에 즉시 반영된다
2. 필수 입력값 누락 시 필드별 오류 메시지가 표시된다
3. 로그인 실패 시 오류 메시지와 재시도 동작이 제공된다
4. 네트워크 장애 시 재시도 버튼이 노출된다

## 4) 릴리즈 전 체크리스트
- [ ] 핵심 시나리오 통과
- [ ] 주요 에러 시나리오 통과
- [ ] 회귀 테스트 완료
