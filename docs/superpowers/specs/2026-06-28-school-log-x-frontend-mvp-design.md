# School Log X 프론트엔드 MVP 설계서

## 1. 목표와 범위
- 목표: 고등학생이 과제/시험 일정을 한눈에 관리하고 누락을 줄이도록 돕는 프론트엔드 MVP 제공
- 포함 범위:
  - FR-001 일정 CRUD
  - FR-002 통합 캘린더(월/주/일)
  - FR-003 D-day 알림(오늘 이후 모든 일정)
  - FR-004 과목별 진행률(0~100) 확인/수정
- 제외 범위: 로그인/회원관리, 서버 연동, 성적 통계

## 2. 기술 및 아키텍처
- 기술: React + Vite
- 구조: 단일 페이지(App 중심) + 기능 컴포넌트 분리
- 상태 관리: `useState` 기반 로컬 상태
- 영속화: 브라우저 `localStorage`

### 2.1 컴포넌트 구성
- `App`
  - 전역 상태(`events`, `subjectsProgress`, `viewMode`, `editingId`, `uiError`)
  - 저장/로딩 로직(localStorage)
  - 검증 및 사용자 안내 메시지 제어
- `ScheduleForm`
  - 일정 등록/수정 폼
  - 필수: 유형(과제/시험), 과목, 제목, 날짜
  - 날짜 입력은 연/월/일 분리 필드(연도 4자리, 월/일 최대 2자리)로 제공
  - 날짜 기본값은 앱 실행일(당일)
  - 선택: 메모
- `CalendarView`
  - 월/주/일 탭 전환
  - 월/주/일 모두 달력 그리드 형태로 렌더링
- `DdayPanel`
  - 오늘 이후 일정 정렬 후 D-day 카드 표기
- `ProgressPanel`
  - 과목별 진행률 표시/수정(0~100)

## 3. 데이터 모델

### 3.1 일정(Event)
```ts
type EventItem = {
  id: string
  type: 'assignment' | 'exam'
  subject: string
  title: string
  date: string // YYYY-MM-DD
  memo?: string
  createdAt: string // ISO
  updatedAt: string // ISO
}
```

### 3.2 과목 진행률
```ts
type SubjectProgress = Record<string, number> // 0~100
```

## 4. 핵심 동작 흐름
1. 앱 시작
   - localStorage에서 `events`, `subjectsProgress` 로드
   - 실패 시 오류 배너 표시 후 빈 상태로 시작
2. 일정 등록/수정/삭제
   - 입력 검증 통과 시 상태 반영
   - 반영 즉시 localStorage 저장
   - 저장 성공 메시지 표시
3. 캘린더 보기 전환
   - 월/주/일 탭 클릭으로 `viewMode` 변경
   - 선택한 보기(월/주/일)에 맞는 달력 그리드로 일정 재구성 후 표시
4. D-day 표시
   - 오늘 이후 일정만 추출
   - 날짜 오름차순 정렬
   - `D-0`, `D-1`, `D-2 ...` 형태로 표기
5. 진행률 수정
   - 과목 카드에서 값 수정
   - 0~100 범위 검증 후 즉시 반영/저장

## 5. 검증 및 오류 처리
- 필수값 누락: 필드별 오류 메시지 + 저장 차단
- 날짜 입력 오류: 날짜 필드 오류 메시지(`날짜를 확인해주세요.`) + 저장 차단
- 진행률 범위 오류: 0~100 범위 안내 + 저장 차단
- localStorage 예외: 상단 오류 배너 + 재시도 안내
- 치명적 렌더링 문제: 사용자 안내 문구 표시

## 6. 접근성
- 모든 주요 조작 요소를 키보드로 접근 가능해야 함
- 입력 요소에 라벨 명시적으로 연결
- 오류 메시지는 `aria-live` 영역으로 전달
- 탭 전환 버튼은 현재 상태를 시각/보조기기 모두 인지 가능하게 표시

## 7. 테스트 전략(프론트엔드 단위)
- 기존 Vitest + Testing Library 유지
- 필수 시나리오:
  - 일정 CRUD 동작
  - 월/주/일 보기 전환
  - D-day 정렬 및 표기
  - 진행률 수정 반영
  - 입력 오류 시 저장 차단 및 메시지 표시

## 8. 완료 기준
- 문서의 FR-001~004를 UI에서 재현 가능
- 오류 발생 시 사용자 안내 메시지 제공
- 핵심 시나리오 중심 테스트가 통과
