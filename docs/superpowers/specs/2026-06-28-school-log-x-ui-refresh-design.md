# School Log X UI Refresh 설계서 (Apple/Toss 느낌)

## 1) 목표
- 현재 기능(일정 CRUD, 캘린더, D-day, 진행률)은 유지
- 시각 완성도를 “밝고 깔끔한 프리미엄” 톤으로 개선
- 앱 사용성을 해치지 않으면서 더 모던한 인상 제공

## 2) 범위
### 포함
- `src/App.jsx`의 레이아웃 래퍼 구조 정리(필요 최소)
- `src/components/*`의 마크업 클래스 정리(필요 최소)
- `src/styles/app.css` 전면 리디자인
- 포커스/오류/키보드 접근성 스타일 유지 및 개선

### 제외
- 비즈니스 로직 변경
- 데이터 모델/저장 방식 변경(localStorage 유지)
- 새로운 기능 추가

## 3) 디자인 방향
- 밝은 화이트 중심 배경 + 은은한 블러 카드
- 여백이 넉넉한 카드형 레이아웃
- 색상은 중립 톤 + 단일 블루 포인트
- 과한 애니메이션 없이 짧고 자연스러운 상태 변화만 사용

## 4) 시각 시스템
### 컬러
- 배경: `#F6F7FB` ~ `#FFFFFF` 그라데이션
- 텍스트(제목): `#111827`
- 텍스트(본문): `#4B5563`
- 포인트: `#3B82F6`
- 경고/오류: `#D14343` (접근성 대비 고려)

### 카드
- 반투명 화이트 배경(`rgba(255,255,255,0.78)`)
- `backdrop-filter: blur(10px)` 적용
- 라운드 16~20px
- 경계는 옅은 반투명 보더
- 그림자는 낮은 강도로 2단 레이어

### 타이포/간격
- 제목 계층 명확화(H1/H2 대비 강화)
- 8px 스케일 간격(8/12/16/24/32) 통일
- 폼 요소 높이와 내부 여백 확대(터치 편의)

## 5) 레이아웃 구조
- 메인 컨테이너: 중앙 정렬 + 최대 폭 제한
- 섹션 배치:
  1. 일정 등록/수정
  2. 캘린더
  3. D-day
  4. 과목별 진행률
- 반응형:
  - 기본: 단일 컬럼
  - 넓은 화면(예: 1024px 이상): 일부 영역 2열 배치 가능(시각만 개선, 정보 구조 동일)

## 6) 인터랙션 원칙
- hover: 밝기/그림자 소폭 증가
- focus-visible: 기존 접근성 유지 + 브랜드 포인트 색 강조
- transition: 120~180ms 이내의 짧은 easing
- 버튼 상태(기본/hover/active/disabled) 명확 분리

## 7) 접근성/안정성
- 현재 라벨 연결 구조, 오류 표시(`role="alert"`), 상태 안내(`role="status"`) 유지
- 텍스트 대비(최소 WCAG AA 수준) 만족하도록 색상 설정
- role/aria 의미를 깨는 마크업 변경 금지

## 8) 테스트/검증 전략
- 기존 테스트(`src/App.test.jsx`, `src/utils/date.test.js`) 모두 통과 유지
- UI 리프레시는 테스트 쿼리(텍스트/role 기반)에 영향 없도록 진행
- 스타일 변경으로 기능 테스트 실패 시, 마크업 최소 수정으로 복구

## 9) 완료 기준
- 화면 인상이 “밋밋한 기본 스타일”에서 “모던 카드형 UI”로 명확히 개선
- 기능 동작 동일성 유지(기존 테스트 통과)
- 접근성 관련 기존 동작(오류/포커스/키보드)이 손상되지 않음
