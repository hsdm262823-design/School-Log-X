# School Log X 간단 프론트엔드 + Azure 배포 설계서

## 1. 목표
- 이 저장소에 **아주 간단한 프론트엔드**를 추가한다.
- 화면은 `제목 + 한 줄 설명 + 상태 문구`만 포함한다.
- Azure Static Web Apps(SWA)로 배포하고, GitHub Actions 자동 배포를 연결한다.

## 2. 범위
### 포함
- React + Vite 기반 최소 프론트엔드 구조
- 단일 페이지 UI(랜딩 화면)
- SWA 배포용 GitHub Actions 워크플로
- 빌드 검증(`npm run build`)

### 제외
- 백엔드 API 연동
- 인증/권한 기능
- 데이터베이스 연동

## 3. 아키텍처
- 클라이언트: React 단일 페이지 애플리케이션
- 배포: Azure Static Web Apps
- CI/CD: GitHub Actions (main 브랜치 푸시 시 자동 배포)

## 4. 구성요소
- `App` 컴포넌트 하나에서 화면을 렌더링한다.
- 내부 상수로 아래 텍스트를 관리한다.
  - 제목
  - 설명
  - 상태 문구
- 서버 호출 없이 정적 렌더링만 수행한다.

## 5. 데이터 흐름
1. 브라우저가 정적 리소스를 로드한다.
2. React 앱이 시작된다.
3. 상수 텍스트를 화면에 렌더링한다.

## 6. 오류 처리
- 앱은 정적 페이지이므로 런타임 오류 지점이 제한적이다.
- 배포 오류는 GitHub Actions 로그에서 즉시 확인한다.
- SWA 토큰/리소스가 없으면 배포를 실패시켜 문제를 명확히 노출한다.

## 7. 테스트/검증
- 로컬: `npm run build` 성공
- 배포 후: SWA URL에서 제목/설명/상태 문구 노출 확인

## 8. 운영 메모
- GitHub 리포지토리 Secret에 `AZURE_STATIC_WEB_APPS_API_TOKEN`이 필요하다.
- SWA 리소스를 새로 생성하고 리포지토리와 연결해야 한다.
